Quora Question Pairs


GOAL


Quora(쿼라)는 질문을 하고 다른 사용자들로부터 답을 받을 수 있는 서비스
이 서비스의 질문들 중에서 어떤 질문이 서로 유사한지 판단하는 모델을 만드는 것
(중복된 질문들을 잘 찾기만 한다면, 이미 잘 작성된 답변들을 사용자가 참고 가능)


DATA







[데이터 분석]


train data의 데이터 구성 형태는
다음과 같다


학습 데이터의 전체 데이터 개수는 다음과 같다
전체 학습 데이터 개수 : 404290


다음으로,
사용할 데이터의 크기를 확인해보자

일반적인 데이터 크기와는 달리
평가 데이터가 훈련 데이터보다 약 5배 큰 것을 알 수 있다
Quora의 경우, 질문에 대한 데이터 수가 적다면
각각 검색해 중복을 찾는 편법 사용이 가능한데,
이를 방지하기 위해 컴퓨터가 만든 질문쌍을 임의적으로 추가했기 때문이다


본격적으로 분석에 들어가보자
분석은 아래와 같은 순서로 진행된다

[데이터 분석]

질문 중복 분석
라벨 빈도 분석

단어 분석


[질문 중복 분석]

우선 각 질문을 리스트로 만든 뒤,
하나의 Series data type으로 합친다


질문 쌍의 수는 40만 개이며,
각각 질문이 두 개씩 있으므로 대략 80만 개의 질문이 있다


다음으로 질문들의 중복 여부 확인을 위해
중복을 제거한 총 질문 수와 반복해서 나타나는 질문의 수를 비교한다


80만 개의 데이터에서 53만 개가 unique data이므로,
27만 개가 중복되어 있음을 알 수 있고
27만 개의 데이터는 11만 개의 고유한 질문으로 이루어져 있음을 알 수 있다


이를 histogram으로 확인해보자


결과를 살펴보면
유일한 질문(중복 횟수가 1)이 가장 많고
대부분의 질문이 중복 횟수가 50번 이하 임을 알 수 있다
매우 큰 빈도를 가진 질문은 이상치가 될 것이다


질문의 중복 분포를 통계치로 수치화해 확인해보자


중복이 발생하는 횟수의 평균이 1.5라는 것은
많은 데이터가 최소 1개 이상 중복되어 있음을 의미한다
즉, 중복이 많음을 알 수 있다


데이터의 직관적 이해를 위해,
중복횟수 box plot을 그려보자


이상치가 너무 넓고 많이 분포해 다른 값을 확인하기가 어렵다


이제 데이터에 어떤 단어가 많이 나오는지 확인하기 위해
WordCloud를 그려보자


best, way, good, difference 등의 단어들이
질문을 할 때 일반적으로 가장 많이 사용 됨을 알 수 있다



[라벨 빈도 분석]

이제 데이터 라벨 ‘is_duplicate’에 대해 알아보자
라벨에는 1(질문이 중복)과 0(질문이 중복이 X)의 경우가 존재했다
라벨들의 횟수 그래프를 그려보자


총 40만 개의 데이터에서
중복 X 데이터가 25만 개,
중복인 데이터가 15만 개이다
이 경우, 중복이 아닌 데이터에 의존도가 높아지면서
데이터가 한 쪽 라벨로 편향되게 되어 학습이 원활히 진행되지 않는다
따라서 최대한 라벨의 개수를 균형 있게 맞춰준 후 진행하도록 하자
(많은 수의 데이터를 줄임 / 적은 수의 데이터를 늘림)



[문자 분석]

다음은 문자 단위로
텍스트 데이터 길이를 분석해보자


[문자 단위 분석]


데이터의 각 질문의 길이 분포는 15 ~ 150에 대부분 모여있으며,
길이가 150에서 급격히 감소하는 것을 볼 때
쿼리의 질문 길이 제한이 150 정도임을 추측할 수 있다


여러가지 통곗값을 확인해보자

최댓값이 평균, 중간값과 매우 큰 차이를 보인다
값이 너무 큰 데이터는 제외하고 학습하자



데이터의 질문 길이 값 박스 플롯 그래프의 분포를 보면
문자 수의 이상치 데이터가 너무 많아 다른 값 확인이 어렵다



[단어 분석]

각 데이터의 단어 개수를 하나의 단위로 사용해 길이 값을 분석한다
단어를 나누는 기준은 띄어쓰기로 정의한다


띄어쓰기를 기준으로 나눈 단어의 개수를 정의하고
히스토그램을 그려보았다


대부분 10개 정도의 단어로 구성된 데이터가 가장 많음을 알 수 있다
데이터의 단어 개수에 대해서 통곗값을 확인해보면



문자 단위 길이 통계와 비슷한 양상을 보인다
최댓값이 중간값,평균에 큰 차이를 보인다
박스플롯으로 데이터 분포를 확인해보자

이상치가 넓고 많이 분포되어 있음을 알 수 있다


몇 가지 특정 경우에 대한 비율을 확인해보자
특수 문자 중 구두점, 물음표, 마침표가 사용된 비율과
수학 기호가 사용된 비율, 대/소문자의 비율을 확인해보자


대부분의 질문들이 물음표와 대문자로 된 첫문자를 가지고 있음을 알 수 있다
보편적인 특징은 제거한다




[전처리]


우선, 필요한 라이브러리와 학습 데이터를 불러오자

먼저 라벨 개수의 균형을 맞춘다
중복이 아닌 데이터의 개수가 더 많기 때문에 해당 데이터 개수를 줄인다

개수가 많은 데이터를
[중복 질문의 중복이 아닌 질문에 대한 비율]로 샘플링하면
데이터 간의 개수가 동일해진다

라벨에 따라 나눠진 데이터를 다시 하나로 합친다
(개수가 동일해진 데이터를 사용하면 균형있는 학습 가능)

비율을 맞춘 데이터를 활용해 데이터 전처리를 진행한다
먼저 문장 문자열에 대한 전처리를 먼저 수행한다


학습 데이터의 질문 쌍을 하나의 질문 리스트로 만들고,
정규 표현식을 사용해 구두점 및 기호를 제거한 뒤
모든 문자를 소문자로 바꿔준다

다음으로 토크나이징을 진행한다
두 질문에 대해 토크나이징 방식을 동일하게 진행하고,
두 질문을 합쳐
전체 단어 사전을 만들기 위해


토크나이징 객체를 만들 때에는
두 질문 텍스트를 합친 리스트에 대해 적용하고,
토크나이징은 해당 객체를 사용해
각 질문에 대해 따로 진행한다

모델 적용을 위해 패딩과정을 거친다
최대 길이 보다 긴 질문은 자르고,
짧은 질문의 부족한 부분은 0으로 채워준다


전처리가 끝난 데이터를 저장하기 전,
라벨값과 단어 사전 저장을 위해 값을 저장한다
단어 사전과 전체 단어의 개수는 딕셔너리 형태로 저장한다

각 데이터를 모델링 과정에서 사용할 수 있도록 저장한다



평가 데이터 전처리

평가 데이터 역시 동일한 방법으로 진행한다





[XGboost Model]


XG 부스트란 ‘eXtream Gradient Boosting’의 약자로
앙상블의 부스팅 기법을 사용하는 방법이다


앙상블 기법이란
여러 개의 학습 알고리즘을 사용해 더 좋은 성능을 얻는 방법으로,
배깅과 부스팅이 존재한다


XG 부스트는 부스팅 기법 중,
Tree Boosting(트리 부스팅) 기법을 활용한 모델이다

Tree Boosting은
여러 개의 의사 결정 트리를 사용하지만 단순히 결과를 평균 내지 않고,
결과를 보고 오답에 대해 가중치를 부여한다
가중치가 적용된 오답이 정답이 될 수 있도록 결과를 만들고
해당 결과에 대한 다른 오답을 찾는 작업을 반복하여 진행한다


XG 부스트는 이러한 트리 부스팅 방식에
경사 하강법을 적용해 최적화 하는 방식이다
의사 결정 트리 구성 시, 병렬 처리를 사용하기 때문에
연산량이 줄어 빠른 학습이 가능하다



본격적으로 XG boost 모델 구현을 해보자!
우선 전처리한 데이터를 불러온다

현재 각 데이터의 두 질문이 따로 구성되어 있는 상태인데
이를 하나씩 묶어 하나의 질문 쌍으로 만든다
즉, 질문[A]와 질문[B]를 [[A],[B]]의 형태로 만들어준다

해당 데이터의 shape를 출력해보면

약 30만 개의 전체 데이터 중,
두 질문이 각각 11개의 질문 길이를 가지고 있음을 확인할 수 있다


다음으로 학습 데이터의 20%를 모델 검증 데이터로 분리한다

XG boost 라이브러리를 불러와 모델을 구현한다
우선, 모델에 적용하기 위해 입력값을 DMatrix 형태로 만든다
(xgboost 모델을 사용하려면 입력값을 xgb라이브러리의 데이터 형식인 DMatrix 형태로 만들어야 함)

모델을 만들고 학습하기 위해 필요한 옵션은 딕셔너리를 만들어 넣어둔다
이후 모델을 생성하고 학습하는 과정을 진행한다


613 스텝에서 학습이 끝났다
이는 더이상 에러가 떨어지지 않아 조기 멈춤한 것이다
이렇게 학습한 모델을 사용해 평가 데이터를 예측하고
예측 결과를 캐글에 제출할 수 있도록 파일로 만든다


우선 전처리한 평가 데이터를 불러온다

불러온 평가 데이터를 XG boost 모델에 적용할 수 있도록
Matrix 형태로 변환 후,
모델의 predict 함수에 적용한다
(predict 함수 error는 모델을 저장했다 불러오면 발생하지 않는다)

예측한 결괏값을 형식에 맞게 파일로 만든다
평가 데이터의 id값과 예측값을 dataframe으로 만든 후
CSV로 저장한다


캐글 제출 결과,
다음과 같은 점수를 받았다

다음에는,
CNN과 MA-LSTM 모델을 활용해보자 :)
