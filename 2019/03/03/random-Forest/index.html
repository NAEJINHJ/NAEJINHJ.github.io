Bag of Words Meets Bags of Popcorn : Random Forest

2019-03-03
BIGDATA, Kaggle, NLP, python



[목차]


Random Forest
Modeling



[Random Forest]


Random Forest에서는
여러 개의 의사결정 트리의 결괏값을 평균낸 것을 결과로 사용한다
이를 통해 분류 혹은 회귀가 가능하다

의사 결정 트리란
트리 구조와 같은 형태로 이루어진 알고리즘으로
각 노드는 하나의 질문이 된다
질문에 따라 다음 노드가 달라지며,
몇 개의 질문이 끝난 후에 결과가 나오는 형태이다

랜덤 포레스트를 통해 강아지 관련 문장을 찾아보자
다음과 같은 두 개의 트리를 만들 수 있을 것이다

실제 랜덤 포레스트 모델에서는 많은 의사결정 트리들이 만들어지고
각 결과들의 평균을 내서 결과를 만든다 :)


[Modeling]


랜덤 포레스트에서는
CountVectorizer를 사용해 모델의 입력값을 만든다
(선형회귀 모델은 TF-IDF와 word2vec 활용해 벡터화한 데이터가 입력값)

CountVectorizer는
텍스트 데이터에서 횟수를 기준으로 특징을 추출하는 방법이다
어떤 단위의 횟수를 셀 것인지는 선택 사항이다
(단위 : 단어, 문자)
보통은 문장을 입력 받아 단어의 횟수를 측정한 뒤 벡터로 만드는 방식을 사용한다


예시를 살펴보자
모듈을 불러오고,
텍스트 데이터 리스트를 설정한 뒤, CountVectorizer 객체를 생성한다

생성한 객체에 fit 함수를 사용해 데이터를 적용하면
자동으로 단어 사전을 생성한다

텍스트 데이터 중 하나만 선택해 벡터로 만들어보자


각 단어가 한 번씩 나왔으므로
해당 단어 사전 순서에 맞게 1 값을 가진다


횟수를 사용해 벡터를 만들기 때문에
직관적이고 간단해서 여러 상황에서 사용할 수 있으나
단순히 횟수만을 특징으로 잡기 때문에
큰 의미가 없지만 자주 사용되는 단어들(조사, 지시대명사)가 높은 특징값을 가지기 때문에
유의미하게 사용하기 어려울 수 있다

본격적으로 모델링에 들어가보자 :)
우선 데이터를 불러온다

불러온 데이터를 특징 벡터로 만들어야 모델 입력값으로 사용할 수 있으므로
CountVectorizer를 불러와 객체를 생성한 뒤
데이터에 적용하자

train_data_features가 어떤 형태인지 출력해보면

(25000x5000) 크기의 행렬로 구성된 것을 확인할 수 있다
이는 25000개의 데이터가 각각 5000개의 특징값을 가지는
벡터로 표현되어 있다는 뜻이다


데이터를 모델에 적용하기 전에
학습 데이터 일부를 검증 데이터로 만들어두자

이제 학습 데이터를 사용해 모델 학습 과정을 수행한다
랜덤 포레스트는
sklearn 라이브러리의 RandomForestClassifier 객체를 사용해 구현한다
해당 모델은 sklearn 라이브러리의 ensemble 모듈안에 있다
모델을 생성한 후 학습 데이터를 적용해 학습시키자 :)

다음으로 검증 데이터로 모델을 검증한다
